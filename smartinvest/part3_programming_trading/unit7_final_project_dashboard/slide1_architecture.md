# 投影片 1/4: 專案藍圖 - 系統架構

**我們要打造一個 Web 應用程式，它的運作流程如下：**

1. **使用者 (瀏覽器)**:
    * 打開我們的儀表板網頁。
    * 在網頁的表單中，輸入想要的策略參數 (例如 MA 的快慢線週期)。
    * 點擊「執行回測」按鈕。

2. **前端 (HTML/JavaScript)**:
    * 將使用者輸入的參數，透過 HTTP POST 請求，發送到我們的後端伺服器。

3. **後端 (FastAPI)**:
    * 接收到請求和參數。
    * 根據參數，動態設定 `backtesting.py` 的策略。
    * 執行回測。
    * 將回測產生的績效報告和資金曲線圖數據，打包好。

4. **前端 (Jinja2)**:
    * 後端呼叫 Jinja2 模板，將打包好的數據「填入」HTML 頁面中。
    * 產生最終的、包含結果的 HTML 網頁。

5. **使用者 (瀏覽器)**:
    * 看到更新後的網頁，上面顯示了績效表格和資金曲線圖。
