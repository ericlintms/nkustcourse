# 單元 7: (總專案) 打造你的交易儀表板

在這個總專案中，我們將綜合運用前面所有學到的知識，打造一個可以執行回測、顯示績效、並呈現結果的網頁儀表板。

## 學習目標

- 整合 `backtesting.py` 與 `FastAPI`。
- 建立一個 API 端點，接收策略參數並執行回測。
- 將回測結果（績效指標、資金曲線圖）傳遞給前端。
- 使用 `Jinja2` 和 `Chart.js`（一個前端圖表函式庫）將結果呈現在網頁上。

### 專案流程

1. **後端 (FastAPI)**:
    - 建立一個 `/backtest` 的 API 路徑。
    - 此路徑接收前端傳來的策略參數 (e.g., MA 週期)。
    - 呼叫 `backtesting.py` 執行回測。
    - 從回測結果中提取需要的績效指標和資金曲線數據。
    - 將這些數據回傳給前端 (或透過 Jinja2 渲染)。
2. **前端 (Jinja2 & HTML/JavaScript)**:
    - 建立一個包含表單的 HTML 頁面，讓使用者可以輸入策略參數。
    - 使用 `fetch` API (JavaScript) 向後端的 `/backtest` 發送請求。
    - 接收到後端回傳的績效數據後，更新網頁上的指標表格。
    - 使用 `Chart.js` 繪製資金曲線圖。
