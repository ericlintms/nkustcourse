# 投影片 1/4: 為什麼需要 API？

## 讓你的策略「活」起來

到目前為止，我們的策略都只是在「歷史資料」上運行的腳本。跑完一次，就結束了。

但如果我們想...

* 建立一個網頁儀表板，**即時** 顯示策略的最新狀態。
* 讓手機 App 可以 **觸發** 一個交易訊號。
* 接收來自交易所的 **即時** 報價，並馬上做出決策。
* 將我們的策略服務提供給其他人使用。

... 我們就需要一個 **持續運行、等待指令、並能回傳結果** 的程式。這就是 **API 伺服器**。

---

## API (Application Programming Interface)

* 應用程式之間的「溝通介面」或「服務窗口」。
* 你不需要知道對方內部的程式碼是如何運作的，只需要知道如何呼叫它的 API，就能取得服務或資料。
* **Web API**: 就是透過網路 (HTTP 協定) 來提供服務的 API。
