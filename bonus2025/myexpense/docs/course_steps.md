# 📚 課程：使用AI Agent開發個人記帳Web系統

## 課程目標

- 學習使用AI Agent輔助程式開發
- 建立簡單但完整的記帳Web應用
- 理解前後端基本架構與資料庫操作

### 課程時長：60分鐘

---

## 📋 課程大綱與提示詞範本

### **第一階段：專案初始化（10分鐘）**

#### 步驟1：建立專案結構

**給Agent的提示詞：**

```md
請幫我建立一個Python FastAPI記帳系統的專案結構，包含：
1. 主程式檔案 main.py
2. templates資料夾（放HTML模板）
3. static資料夾（放CSS）
4. database.py（資料庫設定）
5. models.py（資料模型）
6. requirements.txt（套件清單）

請提供完整的資料夾結構和初始檔案內容。
```

#### 步驟2：安裝必要套件

**給Agent的提示詞：**

```md
我要開發一個FastAPI + Jinja2 + SQLite3的記帳系統，
請給我requirements.txt的內容，包含所有必要套件及版本號。
另外提供pip安裝指令。
```

---

### **第二階段：資料庫設計（10分鐘）**

#### 步驟3：建立資料模型

**給Agent的提示詞：**

```txt
請幫我設計SQLite資料庫的記帳系統資料表，需要記錄：
1. 交易編號（自動遞增主鍵）
2. 日期時間
3. 類型（收入/支出）
4. 類別（餐飲、交通、娛樂、薪資等）
5. 金額
6. 備註

請提供：
1. database.py - 包含SQLite連線設定和建立資料表的函數
2. models.py - 包含Pydantic模型定義
```

---

### **第三階段：後端API開發（15分鐘）**

#### 步驟4：建立FastAPI主程式

**給Agent的提示詞：**

```txt
請建立FastAPI的main.py主程式，需要包含：
1. 首頁路由（顯示記帳清單）
2. 新增記帳的表單頁面路由
3. 處理新增記帳的POST路由
4. 刪除記帳記錄的路由
5. 統計功能（本月收支總計）

使用Jinja2作為模板引擎，所有路由都要能正常運作。
```

#### 步驟5：加入資料庫操作

**給Agent的提示詞：**

```txt
請修改main.py，加入實際的資料庫操作功能：
1. 新增記帳時要寫入SQLite
2. 首頁要從資料庫讀取所有記錄並顯示
3. 刪除功能要真的從資料庫刪除
4. 統計功能要計算資料庫中的實際數據

請提供完整的CRUD操作程式碼。
```

---

### **第四階段：前端介面設計（15分鐘）**

#### 步驟6：建立HTML模板

**給Agent的提示詞：**

```txt
請建立Jinja2的HTML模板檔案，放在templates資料夾：
1. base.html - 基礎模板（包含Bootstrap CDN）
2. index.html - 首頁（顯示記帳清單表格、收支統計）
3. add_record.html - 新增記帳表單

要求：
- 使用Bootstrap 5美化介面
- 表格要顯示所有記帳記錄
- 要有新增和刪除按鈕
- 顯示當月收入、支出、結餘
```

#### 步驟7：優化使用者介面

**給Agent的提示詞：**

```txt
請幫我優化記帳系統的介面：
1. 加入簡單的CSS樣式（在static/style.css）
2. 收入用綠色、支出用紅色顯示
3. 加入圖示（使用Font Awesome）
4. 表單加入輸入驗證
5. 刪除前加入確認對話框

提供完整的CSS和更新後的HTML。
```

---

### **第五階段：功能增強（10分鐘）**

#### 步驟8：加入圖表功能

**給Agent的提示詞：**

```txt
請幫我加入簡單的統計圖表功能：
1. 在首頁顯示本月各類別支出圓餅圖
2. 使用Chart.js實現
3. 需要一個API端點提供JSON格式的統計資料
4. 在index.html加入圖表顯示區域

提供後端API和前端JavaScript程式碼。
```

---

### **第六階段：測試與部署（10分鐘）**

#### 步驟9：建立測試資料與啟動

**給Agent的提示詞：**

```txt
請提供：
1. 建立測試資料的Python腳本（新增10筆範例記帳記錄）
2. 啟動FastAPI伺服器的完整指令
3. 測試各功能的步驟說明
4. 常見問題排除指南
```

---

### 給學生的提醒

1. **使用AI的技巧**
   - 提示詞要具體、清楚
   - 如果結果不理想，調整提示詞重試
   - 可以請AI解釋程式碼的運作原理

2. **除錯技巧**
   - 遇到錯誤訊息，直接貼給AI請求協助
   - 描述「預期結果」vs「實際結果」
   - 一次解決一個問題

3. **延伸學習**
   - 課後可嘗試加入使用者登入功能
   - 可加入預算管理功能
   - 可嘗試部署到雲端平台

---

## 📊 預期成果

完成課程後，學生將擁有：

1. 一個可運作的個人記帳Web系統
2. 使用AI Agent輔助開發的實務經驗
3. 對Web開發架構的基礎認識
4. 持續改進系統的能力和方向

這個課程設計讓學生在有限時間內，透過AI的協助完成一個實用的專案，同時學習到提示工程和基礎的全端開發概念。
